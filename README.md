# ФінПро для дітей

Telegram-бот, який автоматично готує українськомовні пости про гроші,
технології та цифрову безпеку для підлітків. Контент генерує Gemini,
публікації планує APScheduler, а стан і черга модерації зберігаються в
Upstash Redis.

## Основні рубрики

- `#ТехНовини`, `#ШІ_Лайфхак`, `#ВідеоТижня`
- `#ГеймЕкономіка` — гроші та психологія покупок у відеоіграх
- `#СкількиКоштує` — реальні бюджети підліткових цілей
- `#ПершіГроші` — безпечні й легальні перші підробітки
- `#ГрошіВГолові` — психологія покупок і фінансових рішень
- `#СтартапТижня` — реальний продукт, бізнес-модель і ризики
- `#КриптоБезХайпу` — механіки доходу, технології та ризики крипти
- `#ПідпискаПастка` — автосписання, комісії й маркетингові пастки
- `#МіфПроГроші`, `#ФінТруКрайм`, `#ФінКвіз`
- `#ПрофесіїМайбутнього`, `#КіберБезпека`

За замовчуванням автопілот вимкнений: бот надсилає новий матеріал
модератору, а публікація відбувається лише після натискання кнопки.

## Локальний запуск

1. Встановіть Python 3.11.
2. Створіть віртуальне середовище та встановіть залежності:

   ```bash
   python -m venv .venv
   python -m pip install -r requirements.txt
   ```

3. Скопіюйте `env.example` у `.env` і задайте секрети.
4. Запустіть:

   ```bash
   python main.py
   ```

Health endpoint працює на `http://localhost:${PORT:-10000}/`.

### Gemini fallback

Моделі задаються через `GEMINI_MODELS` у порядку пріоритету. Типовий ланцюжок:

1. `gemini-2.5-flash`
2. `gemini-2.0-flash`

При timeout, вичерпаній квоті, недоступній моделі (`403` на конкретну модель)
або невалідному JSON бот автоматично переходить до наступної. Помилки
невірного API-ключа (`401` / `API_KEY_INVALID`) не маскуються fallback-моделями.

## Запуск у Docker

```bash
docker build -t finpro-bot .
docker run --env-file .env -p 10000:10000 finpro-bot
```

Контейнер містить DejaVu Fonts для українського тексту на зображеннях,
працює від непривілейованого користувача та має вбудований healthcheck.

## Northflank

1. Підключіть GitHub-репозиторій у Northflank.
2. Створіть **Combined Service** з типом build `Dockerfile`.
3. Вкажіть порт `10000`, протокол HTTP та health check path `/`.
4. Додайте всі змінні з `env.example` як secrets/configuration.
5. Залиште команду запуску з Dockerfile: `python main.py`.
6. Оберіть одну постійно запущену репліку.

Не запускайте дві репліки одночасно: кожна створить власний scheduler і
може опублікувати один пост двічі. Upstash лишається зовнішнім сховищем,
тому окремий Redis add-on у Northflank не потрібен.

## Команди модератора

- `/autopilot on|off`
- `/status`
- `/pause`
- `/resume`
- `/test crypto` — згенерувати одну рубрику лише в чат модератора
- `/test_all` — послідовно перевірити всі рубрики без публікації в канал
- `/help`

## Перевірка

```bash
python -m unittest discover -v
python -m compileall -q .
```
